CPU 
 -ALU(3장): 각종 산술 연산들과 논리 연산들을 수행하는 회로들로 이루어진 하드웨어 모듈 
 -레지스터 세트(3장) : cpu 내부에 위치한 기억장치, 액세스 속도가 가장 빠름 내부 회로가 복잡해서 지정된 용도로만 사용되는 특수 목적용 레지스터와 적은 수의 일반 목적용 레지스터들만 포함
 -제어 유니트(4장): 명령어를 해석하고, 그것을 실행하기 위한 제어 신호들을 순차적을 발생하는 하드웨어 모듈 CPU 멸령어가 많아질수록 제어 유니트는 복잡해진다.


2-2 명령어 실행
  -명령어를 실행하기 위한 기본적인 내부 레지스터들
	프로그램 카운터(PC):다음에 인출될 명령어의 주소를 가지고 있음
	누산기(AC): 데이터를 일시적으로 저장하는 레지스터(bit 수  = cpu가 한 번에 처리할 수 있는 데이터 비트 =word길이)
	명령어 레지스터(IR):가장 최근에 인출된 명령어가 저장되어 있는 레지스터
	기억장치 주소 레지스터(MAR): PC에 저장된 명령어 주소가 시스템 주소 버스로 출력되기 전에 일시적으로 저장되는 주소 레지스터(주소 버스 선과 직접 접속)
	기억장치 버퍼 레지스터(MBR):기억장치에 저장될 데이터 혹은 기억장치로부터 읽혀진 데이터가 일시적으로 저장되는 버퍼 레지스터(데이터버스 선과 직접 접속)
	기억장치로부터 인출된 명령어는 MBR을 통해 IR에 저장되고 실행 사이클에서 제어유니트로 보내져 해독된다. 데이터는 MBR를 통해 AC에 적재되며 산술 혹은 논리 연산이면 ALU로 보내져 연산이 실행되고 그 결과를 다시 AC에 저장한다. 
	
  -명령어 사이클: cpu가 한개의 명령어를 실행하는 데 필요한 전체 과정
  -명령어 인출: cpu가 기억장치로부터 명령어를 읽어온다.
	인출사이클(fetch cycle): 기억장치의 지정된 위치로부터 명령어를 읽어오는 과정
	마이크로 연산
		MAR <- PC
		MBR <- M[MAR] , PC <- PC + 1
		IR <- MBR
  -명령어 실행: 인출된 명령어를 실행
	실행 사이클(excution cycle): CPU가 인출된 명령어 코드를 해독하고 그 결과에 따라 필요한 연산을 수행
		연산종류
		  -데이터 이동: CPU와 기억장치 간 혹은 CPU I/o장치 간에 데이터 이동
		  -데이터 처리: 데이터에 대하여 산술 혹은 논리 연산 수행( ALU)
		  -데이터 저장: 연산결과 데이터 혹은 입력장치로 읽어들인 데이터를 기억장치에 저장
		  -프로그램 제어: 프로그램의 실행 순서 결정
  	
	
  -인터럽트: 프로그램을 순차적으로 수행하는 도중에 외부로부터 인터럽트 요구가 들어오게 되면 인터럽트 사이클 수행(ISR)
		1. 현재 PC 내용을 스택에 저장 (인터럽트 처리 후 복귀할 주소)
		2. ISR호출을 위해 그 루틴의 시작 주소를 PC에 적재
	인터럽트 사이클: PC에 ISR의 시작 주소를 적재하는 과정

  -간접 사이클: 인출 사이클과 실행 사이클 사이에 위치하고 명령어에 해당하는 데이터를 인출하려고 봤더니 주소가 있어서 한번 더 주소에 접근해야함

2.4 명령어 세트
  -연산 종류 : 
	데이터 이동 
	연산(3장)
	저장
	 프로그램 제어: 명령어 실행 순서를 변경하는 연산들이 필요(분기와 서브루틴 호출)
		-CALL 명령어: 서브루틴을 호출하는 명령어
		-RET명령어: 서부루틴으로부터 원래 프로그램으로 복귀 시키는 명령어
  -데이터 형태(3장) 

  -명령어 형식:필드의 수와 배치 방식 및 각 필드에 포함되는 비트 수를 정의한 것
	연산코드: 수행될 연산을 지정해준다.
	오퍼랜드: 연산에 수행하는데 필요한 데이터 혹은 데이터의 주소를 나타낸다.
	다음 명령어 주소: 순차실행에 필요 x (PC가 있기 때문), 분기 혹은 호출 명령어와 같이 실행순서 변경 때 필요 (PC를 저장하기 위해)
	1주소 명령어, 2주소 명령어 , 3주소 명령어
  -주소지정 방식
	-직접 주소지정 방식
		EA = A
	-간접 주소지정 방식
		EA =(A)
	-즉시 주소지정 방식
	-레지스터 주소지정 방식
		EA = R
	-레지스터 간접 주소지정 방식
		EA =(R)
	-상대 주소지정 방식
		PC+A
	-인덱스 주소지정 방식:Array 사용할 때
